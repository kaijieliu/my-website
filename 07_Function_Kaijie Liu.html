What Is a Function?

A function in Python is a reusable block of code that performs a specific task.

You can think of a function like a machine:

  • You give it input (ingredients).

  • It does some work (a recipe).

  • It may give you back output (a finished dish).

In Python, we:

  1. Define a function (tell Python what it should do).

  2. Call a function (tell Python to actually do it).

A Simple Example

# Defining the function
def say_hello(name):   
  print(f"Hello, {name}!")

# Calling the function  
say_hello("Kitty")
Hello, Kitty!

What a docstring is used for?

1. Explains what the function/class/module does

When someone reads your code, they can quickly understand its purpose without reading the entire implementation.

2. Helps other developers (and your future self!)

  • Why does the function exist?

  • What do the parameters mean?

  • What does it return?

  • What assumptions does it make?

3. Shows up in help() automatically

See an example below. In short, a docstring describes what your code does so people (and tools) can understand and use it without digging into the implementation.

def add(a, b):
    """Add two numbers."""
    return a + b
  
help(add)  # show help()
Help on function add in module __main__:

add(a, b)
    Add two numbers.
print(add(2, 95)) # call the function
97

Parameter and Argument

In the example above, notice how information is passed into the function. The function is defined with a parameter called name, written inside the parentheses on the def line.

def say_hello(name):

This tells Python that the function needs some information in order to do its job - in this case, a name. Because the function expects this value, we must provide it when we call the function:

say_hello("Kitty")

Here, “Kitty” is the argument we pass into the function. When the function runs, that argument is assigned to the parameter name, and Python prints:

Hello, Kitty!

This pattern - define a function with parameters, then call it with arguments - is fundamental to making reusable, flexible code.

Passing Arguments

When a function has multiple parameters, the function call must provide multiple arguments. Python offers several ways to pass those arguments, depending on how explicit or flexible you want to be. The main categories are:

  • Positional arguments – values matched to parameters by order

  • Keyword arguments – values matched using parameter names

Positional arguments

When you call a function, Python matches each argument to a parameter based on its position in the function call. This means the first argument goes to the first parameter, the second argument goes to the second parameter, and so on. In other words, when a function has more than one parameter, Python assigns values to those parameters in order, unless instructed otherwise. These ordered inputs are called positional arguments.

See the example here: we define a function to report students’ test scores.

def report_scores(student_name, subject, score):  # 3 parameters here
  print(f"{student_name} did a {subject} test and scored {score} out of 100 points.")
  
report_scores("Kitty", "Math", 96) # inputing the 3 arguments in order
Kitty did a Math test and scored 96 out of 100 points.

We can call the function as many times as we want.

  
report_scores("Kitty1", "Math", 96)
Kitty1 did a Math test and scored 96 out of 100 points.
report_scores("Kitty2", "Reading", 89)
Kitty2 did a Reading test and scored 89 out of 100 points.
report_scores("Kitty3", "Science", 92)
Kitty3 did a Science test and scored 92 out of 100 points.

And the order matters in positional arguments.

report_scores("Math", "Kitty1", 96) # this doesn't make sense
Math did a Kitty1 test and scored 96 out of 100 points.

Keyword Arguments

A keyword argument is a name–value pair that we pass directly to a function. Instead of relying on the order of the arguments, you specify which parameter should receive each value. This removes any ambiguity and makes function calls easier to read.

Keyword arguments are especially helpful when a function has multiple parameters or when you want your code to clearly express the meaning of each value. Keyword arguments also free us from having to worry about the correct ordering of the arguments.

Let’s see how that works using the report_scores() function defined above.

# see how the arguments don't need to be in the order of student_name, subject, score

report_scores(subject = "Math", student_name = "Kitty1", score = 96) 
Kitty1 did a Math test and scored 96 out of 100 points.
report_scores( score = 96, subject = "Math", student_name = "Kitty1") 
Kitty1 did a Math test and scored 96 out of 100 points.

Default Values for Parameters

When writing a function, you may want some parameters to have predefined values. Python lets you specify a default value for any parameter in the function definition. If you provide an argument for that parameter when calling the function, Python uses your supplied value. If you leave the argument out, Python uses the default value instead.

Default values make your function calls simpler and help clarify the “typical” or most common way the function will be used.

One important rule though - parameter without a default should not follow parameter with a default! Let’s first see what will happen in the followings.

def report_scores(student_name, subject = "Math", score):  # default subject value = "Math"
  print(f"{student_name} did a {subject} test and scored {score} out of 100 points.")
  
report_scores("Kitty", "Reading", 96) 

report_scores("Kitty", 96) 
parameter without a default follows parameter with a default (<string>, line 2)

The above syntax has an error. In a function definition, all parameters without default values must come first, and parameters with default values must come after them. So to correct for the syntax error above, we need to make changes as follows.

def report_scores(student_name, score, subject = "Math"):  # put parameters with default values to the end
  print(f"{student_name} did a {subject} test and scored {score} out of 100 points.")

report_scores("Kitty", 96) # now, this should run!
Kitty did a Math test and scored 96 out of 100 points.

Equivalent Function Calls

Because Python allows positional arguments, keyword arguments, and default parameter values, you often have several different—but equally valid—ways to call the same function. This flexibility means you can choose the style that is clearest and most convenient for your situation. Let’s look at how these approaches can all produce the same result.

# still using the example with default parameter value
def report_scores(student_name, score, subject = "Math"):  
  print(f"{student_name} did a {subject} test and scored {score} out of 100 points.")
  
# use only positional arguments
report_scores("Kitty1", 95)
Kitty1 did a Math test and scored 95 out of 100 points.
report_scores("Kitty1", 95, "Science")
Kitty1 did a Science test and scored 95 out of 100 points.
# use keyword arguments
report_scores(student_name="Kitty1", score=92, subject="Reading")
Kitty1 did a Reading test and scored 92 out of 100 points.
report_scores(score=92, student_name="Kitty1", subject="Reading") # again, order doesn't matter using keyword arguments
Kitty1 did a Reading test and scored 92 out of 100 points.
# mix positional and keyword arguments
report_scores("Kitty1", score=92)
Kitty1 did a Math test and scored 92 out of 100 points.
report_scores("Kitty1", 92, subject="Reading")
Kitty1 did a Reading test and scored 92 out of 100 points.

Return Values

In the above examples, we have been using print() to display the results in the functions. However, this need not be the case. Instead, a function can process data and send a value back to the part of the program that calls it. This value is known as the return value.

A return statement takes a value from inside a function and hands it back to the caller. Returning values is powerful because it lets your functions do the internal “work” (calculations, lookups, processing) while the rest of your program decides what to do with the result.

When you call a function that returns a value, you typically store that value in a variable so you can use it later. Let’s see an example:

# define a function to calculate the average
def calculate_average(score1, score2, score3):
  
    """
    Calculate the average of three numeric scores.

    Parameters
    ----------
    score1 : float or int
        The first score.
    score2 : float or int
        The second score.
    score3 : float or int
        The third score.

    Returns
    -------
    float
        The average (arithmetic mean) of the three scores.
    """

    average = (score1 + score2 + score3) / 3
    return average

# call the function and store the value
avg_score = calculate_average(91, 93, 87)

# print it
print(avg_score)
90.33333333333333
# or use it for other calculations
rescaled_avg_score = round((avg_score - 50)/100, 2)
print(rescaled_avg_score)
0.4

Returning a Dictionary

A function can return any type of value we need - even complex data structures such as lists, tuples, and dictionaries. Returning a dictionary is especially useful when you want your function to send back a collection of related information rather than just a single number or string.

Dictionaries let us label pieces of data with keys, which makes the returned value easier to understand and use. See an example below

# define the function
def build_score_report(student, scores):
    """Return a dictionary containing a student's assessment summary."""
    average = sum(scores) / len(scores)
    
    report = {
        "student": student,
        "scores": scores,
        "average": average,
        "number of assessments": len(scores)
    }
    
    return report
  
# call the function
build_score_report(student = "Kitty1", scores = [78, 85, 89])
{'student': 'Kitty1', 'scores': [78, 85, 89], 'average': 84.0, 'number of assessments': 3}
build_score_report(student = "Kitty2", scores = [92, 89, 82, 94, 76])
{'student': 'Kitty2', 'scores': [92, 89, 82, 94, 76], 'average': 86.6, 'number of assessments': 5}
# we can also call the function and store the return values
report1 = build_score_report(student="Kitty1", scores=[78, 85, 89])
report2 = build_score_report(student="Kitty2", scores=[92, 89, 82, 94, 76])

# so that we can access the dictionary keys
print(report1["student"])
Kitty1
print(report1["number of assessments"])
3
print(report2["student"])
Kitty2
print(report2["number of assessments"])
5

Passing a List

Sometimes we want a function to work with a group of values rather than just a single number or string. In Python, you can pass a list to a function so the function can process many items at once. This is especially useful in educational settings—for example, when analyzing a student’s collection of test scores.

Functions can do many useful things with lists: - loop through the list - compute statistics - modify the list - summarize the list into a return value

Let’s look at some examples.

def print_scores(student, scores):
    """Print each of a student's assessment scores."""
    print(f"Scores for {student}:")
    for score in scores:
        print(f" - {score}")

print_scores("Kitty", [92, 89, 82, 94, 76])
Scores for Kitty:
 - 92
 - 89
 - 82
 - 94
 - 76
def summarize_scores(scores):
    """Return a summary dictionary for a list of scores."""
    summary = {
        "scores":  scores,
        "total":   sum(scores),
        "average": sum(scores) / len(scores),
        "highest": max(scores),
        "lowest":  min(scores)
    }
    return summary


summary = summarize_scores([78, 85, 89, 92])

print(summary)
{'scores': [78, 85, 89, 92], 'total': 344, 'average': 86.0, 'highest': 92, 'lowest': 78}
def add_score(scores, new_score):
    """Add a new score to a student's list of scores."""
    print("Scores before updating:", scores)
    scores.append(new_score)
    print("Scores after updating:", scores)


student_scores = [78, 85, 89]
add_score(student_scores, 92)
Scores before updating: [78, 85, 89]
Scores after updating: [78, 85, 89, 92]
def modified_scores(scores):
    """Add 5 extra points to each score (modifies the original list)."""
    print("Scores before updating:", scores)
    for i in range(len(scores)):
        scores[i] += 5
    print("Scores after updating:", scores)

modified_scores([78, 85, 89])
Scores before updating: [78, 85, 89]
Scores after updating: [83, 90, 94]

However, modifications to a list should use caution. Or, we can avoid direct modification by making a copy.

def modified_scores2(scores):
    """Add 5 extra points to each score (modifies the original list)."""
    print("Scores before updating:", scores)
    new_scores = scores.copy() # making a copy
    for i in range(len(new_scores)):
        new_scores[i] += 5
    print("Original scores:", scores)
    print("Updated scores:", new_scores)

modified_scores2([78, 85, 89])
Scores before updating: [78, 85, 89]
Original scores: [78, 85, 89]
Updated scores: [83, 90, 94]

Using a Function with an if-elif-else Statement

We will see how through an example:

def grade_with_feedback(score):
    """Return a dictionary containing a letter grade and written feedback."""
    
    if score >= 90:
        grade = "A"
        feedback = "Excellent performance."
    elif score >= 80:
        grade = "B"
        feedback = "Strong work. Keep it up!"
    elif score >= 70:
        grade = "C"
        feedback = "Satisfactory. Some improvement needed."
    elif score >= 60:
        grade = "D"
        feedback = "Below expectations. Review the material again."
    else:
        grade = "F"
        feedback = "Needs significant improvement."
    
    return {
        "grade": grade,
        "feedback": feedback
    }
    
grade_with_feedback(score = 95)
{'grade': 'A', 'feedback': 'Excellent performance.'}
grade_with_feedback(score = 65)
{'grade': 'D', 'feedback': 'Below expectations. Review the material again.'}
grade_with_feedback(score = 13)
{'grade': 'F', 'feedback': 'Needs significant improvement.'}

Using a Function with a while Loop

A while loop lets you keep calling a function as long as a certain condition remains true. This is useful when: - we want to repeatedly ask a user for input - we want to process multiple sets of data - we don’t know ahead of time how many times you’ll need the function

A while loop keeps running until the condition becomes False.

Here’s a function that builds a dictionary for a single student’s score entry:


def record_score(student, score):
    """Return a dictionary representing one student's score entry."""
    return {
        "student": student,
        "score": score
    }

Now let’s use a while loop to let the user enter multiple students’ scores.

all_records = []   # list to store all score entries
active = True      # loop flag

while active:
    print("\nEnter a student's score:")
    
    name = input("Student name (or 'q' to quit): ")
    if name == 'q':
        active = False
        continue

    score = int(input("Score: "))
    
    record = record_score(name, score)  # call the function inside the loop
    all_records.append(record)          # store each result

print("\nAll recorded scores:")
print(all_records)
