The input Function

The input() function is how Python allows your program to receive information directly from the user. When your code reaches an input() call, the program pauses and waits for the user to type something and press ENTER. Whatever the user enters is returned as a string and can be stored in a variable for later use.

The Prompt Argument

The input() function takes one argument: the prompt that tells the user what kind of information to enter.

name = input("What is your name?")  # the variable, name, stores the input value

print(name)  # then, can be printed out

Multi‑Line Prompt for Assessment Instructions

Some assessment instructions require more context, such as explaining the purpose, giving a scenario, or defining how answers will be used. Python allows us to build these multi‑line prompts cleanly by assigning them to a variable. As the example shown below, the += operator lets you build a readable, multi‑line prompt.

instructions = "SECTION: Course Feedback\n"
instructions += "Please respond to the following prompt honestly.\n"
instructions += "Q3: What is one thing your instructor could do to improve your learning experience? "

feedback = input(instructions)
print("Feedback received:", feedback)

Using int() to Accept Numerical Input

Since Python interprets everything the users input as a string, even when a number is entered, it will return as the string representation of the number. If we are trying to use the input number for further mathematical operations, we will run into errors. This can be resolved by using the int() function.

age = input("How old are you?")
print(age)
print(age - 1)  # age is a string, so this results in an error

print(int(age) - 1)  # this works as age is converted into an integer

While Loops

In Python, a for loop iterates over a collection of items, running its block of code once for each item. A while loop works differently: it repeats a block of code as long as a certain condition remains true.

The while Loop in Action

A while loop continually checks a condition. If the condition is True, the loop executes. If the condition becomes False, the loop stops.

See a simple example below, where consecutive integers starting from 1 and on are printed as long as last one is less than 10.

i = 1

while i <10:
  print(i)
  i += 1  # This line increases i by 1 each loop. 
          # Without it, the condition never changes — causing the loop to run forever.

Note: If our program ever gets stuck in an infinite loop, press CTRL-C.

To avoid writing infinite loops, it’s important to test every while loop and make sure the loop stops when expected to. The ways to do this may involve making sure at least one part of the program can make the loop’s condition False or cause it to reach a break statement.

Now, let’s see a more complicated example.

print("SECTION: Study Activities Log")
print("Enter 'done' when you have finished listing your study activities.\n")

activities = []
activity = input("Enter a study activity: ")

while activity.lower() != "done":
    activities.append(activity)
    activity = input("Enter another activity (or type 'done'): ")

print("Activities recorded:", activities)

Use a Flag

A flag is a variable (usually named something like active, running, or continue_program) that acts as a signal to the program:

  • When the flag is True, the program keeps running.

  • When the flag becomes False, the program stops.

This structure lets your main while loop check one clean condition, while the rest of your code handles all the detailed tests that might shut the program down.

Flags make our program:

  • Easier to read
  • Easier to debug
  • Easier to extend when more stop conditions are added
  • Useful for interactive programs (like assessments, surveys, input loops)

For example, the while loop below continues while the flag variable is True (active = True); when the user types “quit”, the flag is set to False (active = False), causing the loop to stop.

active = True     # This flag controls the loop

while active:
    response = input("Enter a comment about your study session (type 'quit' to stop): ")

    if response.lower() == "quit":
        active = False     # Turn the flag off → loop ends
    else:
        print("Recorded:", response)

Use break to Exit a loop

When Python encounters a break inside a loop, it immediately terminates that loop and jumps to the next line of code after the loop.

The break statement is a powerful tool for controlling the flow of a program. It allows us to:

  • Exit a loop at a specific moment
  • Stop further execution of the loop’s body
  • Prevent unwanted code from running
  • Respond to events or inputs in real time

Let’s see a simple example:

i = 1

while i < 10:
    print(i)

    if i == 5:
        break   # Immediately exit the loop when i reaches 5

    i += 1

Let’s see a more complicated example:

responses = []

while True:   # loop runs indefinitely unless break is triggered
    response = input("Enter a comment about your study session (type 'stop' to end): ")

    if response.lower() == "stop":
        break   # exit the loop immediately
      
    responses.append(response)

    
print("Recorded:", responses)

Use continue in a Loop

There are times when we don’t want to exit a loop entirely - we simply want to skip the rest of the current loop iteration and return to the beginning of the loop. Python’s continue statement does exactly that. Instead of stopping the loop (like break), continue tells Python: “Skip the remaining code in this iteration, and jump back to the top of the loop to check the condition again.”

This gives us fine‑grained control over which lines of code run and allows us to ignore inputs or events selectively.

With the use of continue, this example code below prints only odd numbers from 1 to 9, because continue causes the loop to skip the print() line when i is even.

i = 0
while i < 10:
    i += 1

    if i % 2 == 0:   # If i is even
        continue     # Skip printing it

    print(i)

An example with the input() function:

while True:
    response = input("Describe one effective study strategy you used today (leave blank to re-try): ")

    if response.strip() == "":
        print("No response detected — please try again.\n")
        continue      # Skip recording and return to the top of the loop

    print("Recorded:", response)
    break            # End after a valid response

Using a while Loop with Lists and Dictionaries

Although a for loop is effective for looping through a list, we should not modify a list within a for loop because Python will have trouble keeping track of the items inside a list. Instead, we can use a while loop to modify a list as we work through it.

Using while loops with lists and dictionaries allows us do the followings.

Remove items as you process them

Scenario: You have a list of unreviewed survey responses. As each response is “processed,” you remove it from the list.

unreviewed_responses = [
    "I studied in the library.",
    "I watched the lecture recording twice.",
    "I worked with a study group."
]

print("Processing responses...\n")

while unreviewed_responses:
    current_response = unreviewed_responses.pop(0)  # remove first item; blank for default to the last item.
    print("Reviewing response:", current_response)

print("\nAll responses have been processed.")

Add new items as needed

Scenario: You’re logging study activities. As long as the student keeps entering new activities, they’re added to the list.

study_activities = []

print("Study Activities Log")
print("Enter 'done' when you are finished.\n")

while True:
    activity = input("Enter a study activity: ")

    if activity.lower() == "done":
        break

    study_activities.append(activity)  # add new item to the list

print("\nYou reported these activities:")

for a in study_activities:
    print("-", a)

Transfer items from one list to another

Scenario: You have a list of participants who haven’t completed a survey. When they complete it, they are moved to a “completed” list.

incomplete_participants = ["ann", "david", "maria"]
completed_participants = []

print("Sending survey to participants...\n")

while incomplete_participants:
    current = incomplete_participants.pop(0)  # take from incomplete
    print("Survey completed by:", current.title())

    completed_participants.append(current)    # move to completed list

print("\nCompleted participants:")
for name in completed_participants:
    print("-", name.title())

Build dictionaries from repeated user input

Scenario: You’re collecting named responses from multiple participants and want to store them in a dictionary for later analysis.

survey_results = {}  # name -> response

print("Study Preparation Survey")
print("Type 'quit' as the name to stop.\n")

while True:
    name = input("Participant name: ")
    if name.lower() == "quit":
        break

    response = input("How prepared did you feel for your last study session? ")

    survey_results[name] = response  # build dictionary entry
    print("Response recorded for", name + ".\n")

print("\nSurvey summary:")
for participant, answer in survey_results.items():
    print(f"{participant.title()}: {answer.title()}")

Maintain full control over loop conditions

Scenario: You want to keep a survey “session” running while:

the program is active, and the user hasn’t chosen to stop, and they haven’t exceeded a max number of responses.

You can manage that cleanly with a flag and counters.

responses = []
active = True
max_responses = 3
count = 0

print("Short Course Feedback Survey")
print(f"You may give up to {max_responses} comments.\n")

while active:
    comment = input("Enter a comment about the course (or type 'stop'): ")

    if comment.lower() == "stop":
        active = False              # user chose to stop
    elif comment.strip() == "":
        print("Empty comment ignored.\n")
        # continue without changing count
    else:
        responses.append(comment)
        count += 1

        if count >= max_responses:
            print("\nMaximum number of responses reached.")
            active = False          # end due to limit

print("\nRecorded comments:")
for c in responses:
    print(f"-{c.title()}")
