if Statement

What is an if Statement?

An if statement allows our program to make decisions. It checks a condition and runs a block of code only if that condition evaluates to True. At the core of every if statement is a conditional test - an expression that evaluates to either True or False. When the test is True, Python runs the indented code that follows the if. When it’s False, that block is skipped.

Why using if Statement?

if statements help us:

  • Check conditions you care about (e.g., a value, a user input, a number)

  • Run code only in specific situations

  • Handle special cases differently from the general rule

  • Make your program dynamic and responsive

if - else Statements

In programming, we often want to take one action when a condition is True and a different action when the condition is False. Python’s if–else structure allows you to define two distinct code paths:

  • the if block runs when the condition passes

  • the else block runs when the condition fails

This is useful whenever you want to guarantee that one of two outcomes will always occur. In addition, we can also do something interesting when combining lists and the if statements.

For example, you may want to handle most list items in one way but treat a particular value differently.

# consider we have a list of signed-up participants tagged from A to E according to their orders of signing up 
signup_list = ["A", "B", "C", "D", "E"]


# see what an if statement can do
for s in signup_list:
  if s == "A":
    print(f"{s} was the first person signing up.")
  else:
    print(f"{s} was not the first person signing up.")
A was the first person signing up.
B was not the first person signing up.
C was not the first person signing up.
D was not the first person signing up.
E was not the first person signing up.

Conditional Tests

Conditional Tests is the heart of if statements. A conditional test is an expression that Python evaluates as either True or False. The checks involve equality checks and inequality checks.

Eqalilty Checks

Use the equality operator == to check whether two values are the same.

# this creates the `school` variable
school = "Greenwood"
print(school)
Greenwood
# this below only checks if school equals "Greenwood"
school == "Greenwood"
True
# the value of school has not been changed
print(school)
Greenwood

Case Sensitivity

Equality checks are case sensitive. See examples below:


school = "Greenwood"

school = "Greenwood" # True

school = "greenwood" # False

Checking Multiple Conditions

Checking Whether a Value Is in a List

Sometimes your program needs to know whether a particular value already exists in a list before taking action. This is common when you want to:

  • verify that a username is not already taken

  • check whether a student ID appears in a roster

  • confirm whether a location, category, or code has already been recorded

  • validate user input against allowed options

Python provides an easy way to do this through the keyword in.

enrolled_students = ['s101', 's102', 's103']

student_id = 's102'

if student_id in enrolled_students:
    print("Student is already enrolled.")
Student is already enrolled.

Checking Whether a Value Is Not in a List

Sometimes, it’s also important to know if a value does not appear in a list. You can use the keyword not in this situation.

enrolled_students = ['s101', 's102', 's103']

student_id = 's105'

if student_id not in enrolled_students:
    print("Student is not enrolled.")
Student is not enrolled.

Boolean Expressions

A Boolean expression is simply a conditional test - an expression that evaluates to one of two possible Boolean values: True/False. These values are the foundation of decision‑making in Python. Every if statement depends on a Boolean value to determine whether a block of code should run. Whenever we compare values - such as checking equality, inequality, or membership, we create a Boolean expression.

Revisiting the first example we had will hopefully makes it easier to understand Boolean and condition checking better.

signup_list = ["A", "B", "C", "D", "E"]

signup_list[0] == "A" # this generates a boolean value
True
signup_list[1] == "A" # and this, too
False
for s in signup_list: # the for loop calls each element in signup_list
  if s == "A":        # s == "A" is a boolean expression that generates a boolean value; if this is "True", then the following print will be done.
    print(f"{s} was the first person signing up.")
  else:  # if this s == "A" equals "False", then the following will be printed instead
    print(f"{s} was not the first person signing up.")
A was the first person signing up.
B was not the first person signing up.
C was not the first person signing up.
D was not the first person signing up.
E was not the first person signing up.

The if - elif - else Chain

Now, consider we’ll need to test more than two possible situations, which is not uncommon. Here, Python’s if - elif - else syntax will allow us to do that. In the if - elif - else chain, Python will only execute one block in that chain. When we write an if - elif - else chain, Python evaluates each conditional test from top to bottom, in order. As soon as it finds a test that evaluates to True, it runs the code under that condition and skips all remaining tests, even if some of them would also have been True.

The following example shows how the if - elif - else chain works and the order of the execution by setting the score to be 578, then 399.

score = 578    # Student's scaled test score for reading literacy

if score < 400:
    print("Below Baseline")
elif score < 500:
    print("Baseline")
elif score < 600:
    print("Proficient")
else:
    print("Advanced")
Proficient
score = 399    # Student's scaled test score for reading literacy

if score < 400:
    print("Below Baseline")
# see how the lines below were not executed although being true 
elif score < 500:      
    print("Baseline")
elif score < 600:
    print("Proficient")
else:
    print("Advanced")
Below Baseline

As the example shows, we can use as many elif blocks as we like.

Omitting the else Block

The else block runs when none of the previous conditions are met. Because it does not include a condition of its own, it becomes a catchall: any value, scenario, or state that wasn’t explicitly handled by earlier tests will fall into the else block. This can be useful, but it can also introduce risk.

Imagine you’re assigning achievement levels based on a student’s scaled score and there is a value of 9999 indicating missing value. The following code will print out “Advanced” mistakenly.

score = 9999    # Student's scaled test score for reading literacy is missing and coded as 9999

if score < 400:
    print("Below Baseline")
elif score < 500:      
    print("Baseline")
elif score < 600:
    print("Proficient")
else:
    print("Advanced")  # All of the above do not pass 
Advanced
                       # and then, "Advanced" will be printed, 
                       # which generates an error.

We could have omit the else block to avoid this mistake. In that case, nothing will be printed out. In fact, what’s even better is to do the following:

score = 9999

if 0 <= score < 400:
    print("Below Baseline")
elif 400 <= score < 500:
    print("Baseline")
elif 500 <= score < 600:
    print("Proficient")
elif 600 <= score <= 800:
    print("Advanced")
else:
    print("Invalid score: review required.")
Invalid score: review required.

Using Multiple Lists in an if - else Statement

We will demonstrate this using an example below. In this example, we’ll:

  • loop through all students who should be in the assessment

  • check:

    • if they submitted the test

    • if they have accommodations

  • use if - elif - else to decide what message to print for each student

# List of students who are supposed to take the assessment
enrolled_students = ['s101', 's102', 's103', 's104']

# Students who actually submitted the assessment
submitted_students = ['s101', 's103']

# Students who require special accommodations
accommodation_students = ['s103', 's104']

for student_id in enrolled_students:
    if student_id in submitted_students and student_id in accommodation_students:
        print(f"{student_id}: Submitted with accommodations.")
    elif student_id in submitted_students:
        print(f"{student_id}: Submitted without accommodations.")
    elif student_id in accommodation_students:
        print(f"{student_id}: Needs accommodations but did not submit.")
    else:
        print(f"{student_id}: No submission on record.")
s101: Submitted without accommodations.
s102: No submission on record.
s103: Submitted with accommodations.
s104: Needs accommodations but did not submit.
