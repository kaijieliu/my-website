Class

What Is a Class?

A class in Python is a blueprint for creating objects.
It defines:

  • Attributes → the data or state each object will hold

  • Methods → the behaviors or actions each object can perform

Classes are the foundation of object‑oriented programming (OOP).

What is Object‑Oriented Programming (OOP)?

Object‑oriented programming models real-world ideas directly in code. OOP is one of the most effective approaches to writing software. In OOP, we write classes that represent real-world things and situations and create objects based on these classes.

  • When writing a class, we define the general behavior that a whole category of objects can have.

  • When creating individual objects from a class, each object is automatically equipped with the general behavior and can then be given unique traits.

  • Making an object from a class is called instantiation, where we work with instances of a class.

An Example of Creating and Using a Class

Let’s see an example of creating a class and making an instance from a class. In this example, we define a classroom class to represent a real-world classroom. Each classroom will have: - a name (like “Physics 101”) - a teacher - a number of students

We’ll also give the class some methods to describe its behavior.


# Defines a class called Classroom. This is our blueprint.
class classroom:
  
  # __init__() is the constructor method.
  # Python calls __init__() automatically whenever we create a new Classroom object.
  # self.name, self.teacher, self.num_students are attributes, which store the state of each classroom object.
  
  def __init__(self, name, teacher, num_students):
    """
    Initializing a new classroom instance.
    Parameters:
        - name (str): The name of the classroom or course.
        - teacher (str): The teacher in charge of the classroom.
        - num_students (int): How many students are in the classroom.
    """
    self.name = name
    self.teacher = teacher
    self.num_students = num_students
    
    
  # describe() is a method that defines the behaviors of a classroom.
  
  def describe(self):
    """ Print a summary of the classroom. """
    print(
      f"Classroom: {self.name}\n"
      f"Teacher: {self.teacher}\n"
      f"Total enrollment of students: {self.num_students}"
      )
      
  
  # add_student() and remove_student() are the methods that chenge the number of students.
  
  def add_student(self):
        """Increase the number of students by 1."""
        self.num_students += 1

  def remove_student(self):
    """Decrease the number of students by 1, if possible."""
    if self.num_students > 0:
      self.num_students -= 1
    else:
      print("There are no students to remove.")

Now, we instantiate (i.e., cerate objects from) the classroom class and use its method.

# Creating two Classroom instances (objects)
classroom1 = classroom("Physics 101", "Dr. Smith", 30)
classroom2 = classroom("Classical Music", "Ms. Johnson", 25)

In this example, the classroom class represents a real‑world classroom. Each instance (object) of Classroom has:

  • Attributes such as name, teacher, and num_students.

    • The __init__() method receives these values as parameters and assigns them to the instance so each object can store its own data.

  • Methods such as describe(), add_student(), and remove_student().

    • The describe() method prints out the classroom’s current attributes.

    • The add_student() method increases the number of students in the classroom by 1.

    • The remove_student() method decreases the number of students by 1, as long as the classroom still has students.

    • Both add_student() and remove_student() modify the instance’s attributes, demonstrating how methods allow objects to change their internal state.

Inheritance

What Is Inheritance?

Inheritance allows you to create a new class that is a specialized version of an existing class.
Instead of starting from scratch, the new class inherits:

  • attributes

  • methods

  • behavior

from the original class.

The original class is the parent class (also called base class or superclass).

The new class is the child class (also called derived class or subclass).

The child class can:

  • reuse everything in the parent class

  • override parent methods (customize behavior)

  • add brand‑new methods or attributes of its own

This makes inheritance a powerful tool for reducing repetition and building logical class hierarchies.

The __init__() Method in a Child Class

When we create a child class, we often want it to automatically receive all the attributes defined in the parent class.

To do that, the child class should call the parent class’s __init__() method.

This ensures:

  • all parent attributes get created

  • the child class starts with the same initialization process

  • you can add new child‑specific attributes afterward

You call the parent’s __init__() using the super() function, like this:


class Parent:
    def __init__(self, value):
        self.value = value

class Child(Parent):
    def __init__(self, value, extra):
        super().__init__(value)   # Call parent initializer
        self.extra = extra        # Child-specific attribute

The super() function is a special function that allows you to call a method from the parent class.

Now, we want to create a child class: science_C_classroom This child class inherits everything from classroom, but adds a new attribute: the lab availability.


class science_classroom(classroom):
  
  def __init__(self, name, teacher, num_students, has_lab):
    # call super() *as a function*, then .__init__
    super().__init__(name, teacher, num_students)  # call parent __init()__
    self.has_lab = has_lab                         # new child attribute
    
  def describe(self): 
    super().describe()  # reuse the parent behavior
    print(f"Lab availability: {self.has_lab}")
    
    
  def show_lab_status(self):  # define a new method
    if self.has_lab:
      print(f"{self.name} has a fully equipped science lab.")
    else:
      print(f"{self.name} does not have a lab.")

Define a New Attribute, Method, and Override a Method

In the above example: - self.has_lab = has_lab is defining a new attribute in the child class. This attribute does not exist in the parent classroom class. - show_lab_status() is a new method defined only in the child class, adding behavior specific to science_classroom. - The describe() method in science_classroom is overriding the describe() method from the parent class. It first calls super().describe() to reuse the parent behavior, then adds extra information about lab availability.

Let’s see how the child class is used.

# create new science classroom instances/objects
science_classroom1 = science_classroom("Physics 101", "Dr. Smith", 25, has_lab = False)
science_classroom2 = science_classroom("Biology 101", "Dr. Johnson", 20, has_lab = True)

# call the overridden method
